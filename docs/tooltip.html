Tooltip

A lightweight tooltip using the Popover API with CSS Anchor Positioning. No dependency, no lock-in. Just copy paste.

Support: Popover API is broadly available. CSS Anchor Positioning is newer and currently best in Chromium-based browsers.

Placement Variants

ClassPosition
.vui-tooltip--topAbove the trigger (default)
.vui-tooltip--bottomBelow the trigger
.vui-tooltip--leftLeft of the trigger
.vui-tooltip--rightRight of the trigger

Multiple Tooltips

Each tooltip needs a unique anchor name. Set anchor-name on the trigger and position-anchor on the tooltip using inline styles or additional CSS rules.

HTML 
<!-- First tooltip pair -->
<button style="anchor-name: --tip-1" popovertarget="tip-1">A</button>
<div class="vui-tooltip" id="tip-1" popover
     style="position-anchor: --tip-1">Tooltip A</div>

<!-- Second tooltip pair -->
<button style="anchor-name: --tip-2" popovertarget="tip-2">B</button>
<div class="vui-tooltip" id="tip-2" popover
     style="position-anchor: --tip-2">Tooltip B</div>

API

ClassDescription
.vui-tooltipThe tooltip container (popover element)
.vui-tooltip--topPosition above anchor
.vui-tooltip--bottomPosition below anchor
.vui-tooltip--leftPosition left of anchor
.vui-tooltip--rightPosition right of anchor

Notes

CSS Anchor Positioning is a newer specification. As of early 2025, it is supported in Chromium-based browsers (Chrome 125+, Edge 125+). Firefox and Safari support is in development. For broader compatibility, consider a fixed-position fallback.

The tooltip uses the Popover API for show/hide behavior. The popovertargetaction="toggle" attribute controls the interaction model. For hover behavior, a small script adding showPopover() / hidePopover() on mouseenter/mouseleave is needed.
