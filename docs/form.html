Notes

Error messages appear automatically when the user interacts with a required field and leaves it empty, or when the value fails HTML5 constraint validation (e.g. invalid email format). The :user-invalid pseudo-class ensures errors only show after user interaction, not on initial page load.

The browser will also show its native validation tooltip on form submission. To suppress it and rely only on the CSS error states, add novalidate to the <form> element.
